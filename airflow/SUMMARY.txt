## 🚀 Pipeline Architecture

### Daily Execution Timeline
```
00:00 UTC   ┌─ Airflow Scheduler triggers DAG
            │
00:00-00:02 ├─ Task 1: Generate filename (covid_YYYY-MM-DD.csv)
            │
00:02-00:05 ├─ Task 2: Upload CSV to S3 (~3 min for typical file)
            │
00:05-00:06 ├─ Task 3: Build Snowflake COPY SQL
            │
00:06-00:10 ├─ Task 4: Load from S3 → COVID_DB.RAW (~4 min)
            │
00:10-00:11 ├─ Task 5: Data Quality Validation
            │
00:11-00:20 ├─ Task 6: Run dBT (staging → mart tables) (~9 min)
            │
00:20-00:21 ├─ Task 7: Notify Completion & Log
            │
            └─ TOTAL: ~21 minutes (Well under 2-hour SLA) ✓
```

### Data Transformations in Snowflake
```
S3 CSV File                    Snowflake (COVID_DB)
│                             │
├─ Raw data                   └─ RAW.COVID_DATA (raw from S3)
│                                    ↓
│                                STG_COVID_DATA (via dBT stg_covid_data)
│                                    ├─ Cleaned & validated
│                                    ├─ Unique identifiers
│                                    └─ dbt_updated_at
│                                    ↓
│                                FACT_COVID (via dBT fact_covid)
│                                    ├─ Fact table with dimensions
│                                    ├─ Country, province, dates
│                                    └─ Case counts
│                                    ↓
│                                COUNTRY_COVID (via dBT country_covid)
│                                    ├─ Country-level aggregates
│                                    ├─ Latest observations
│                                    └─ Total cases by country
```

## 💡 Usage Examples

### Trigger Pipeline Manually
```powershell
# Immediate trigger
airflow dags trigger covid_batch_pipeline

# Specific date
airflow dags trigger covid_batch_pipeline --exec-date 2026-03-31
```

### View Pipeline Status
```powershell
# Recent runs
airflow dags list-runs covid_batch_pipeline | head -20

# View specific task logs
airflow logs covid_batch_pipeline load_to_snowflake 2026-03-31

# Show DAG structure
airflow dags show covid_batch_pipeline
```

### Configure Variables (if not using setup script)
```powershell
airflow variables set COVID_S3_BUCKET covid-data-bucket
airflow variables set SNOWFLAKE_DATABASE COVID_DB
airflow variables set SNOWFLAKE_SCHEMA RAW
airflow variables set DBT_PROJECT_DIR ../
# ... etc
```

### Via Airflow Web UI
1. Open http://localhost:8080
2. Go to **DAGs** section
3. Click **covid_batch_pipeline**
4. Click **Trigger DAG** button (top right)
5. Watch execution in real-time

## 🔧 Configuration Details

### Connections Required
1. **aws_default** (AWS)
   - Type: aws
   - Access Key: Your AWS key
   - Secret Key: Your AWS secret

2. **snowflake_default** (Snowflake)
   - Type: snowflake
   - Host: account.snowflakecomputing.com
   - User: Your username
   - Password: Your password

### Variables Required
| Variable | Example |
|----------|---------|
| COVID_S3_BUCKET | covid-data-bucket |
| COVID_S3_PREFIX | covid-data |
| LOCAL_COVID_FILE_DIR | C:\data\covid |
| SNOWFLAKE_DATABASE | COVID_DB |
| SNOWFLAKE_SCHEMA | RAW |
| SNOWFLAKE_WAREHOUSE | COMPUTE_WH |
| DBT_PROJECT_DIR | ../ |
| DBT_EXECUTABLE | ../.dbt-venv/Scripts/dbt.exe |

## 🚨 Error Handling

### Automatic Retries
```
Task fails
    ↓
Retry 1 (immediate)
    ↓ (if still fails)
Wait 5 minutes
    ↓
Retry 2
    ↓ (if still fails)
Alert to Slack/Email
Mark task as failed
```

### SLA Management
- Target: 2 hours
- If exceeded: UI alert, email notification
- Configurable per DAG
